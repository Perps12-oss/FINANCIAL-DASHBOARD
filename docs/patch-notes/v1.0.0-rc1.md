# Release notes — v1.0.0-rc1 (release candidate)

**Date:** 2025-03-07 (post–professionalization pass)

---

## Summary

First release candidate after the five-phase repo professionalization. No new product features; structural cleanup, API contract clarity, and documentation alignment.

---

## What changed

- **Repo truth and root:** Root cleaned; audits, plans, dumps, patch notes in `docs/` or `archive/`. `showHelp()` fixed for non-Spreadsheet context; triage matrix added under `docs/`.
- **Backend:** Standardized response envelopes (`ok`, `data`, `error`, `meta`) on core API endpoints; legacy `success` preserved for compatibility. `MAX_TRANSACTIONS_RETURNED` moved to Config.gs.
- **Frontend:** System page load wrapped in try/catch; envelope-aware parsing for health/diagnostics/logs; visible error state when system data fails to load.
- **Docs:** RELEASE.md updated with refactor/* and release/* branch expectations; README, ARCHITECTURE, SCHEMA, ROADMAP already aligned with current structure.

---

## Validation

- Root contains only app source, README, .gitignore, .claspignore, appsscript.json.
- Core endpoints return consistent envelope; Code.gs remains entrypoints-only; Data.gs is the single data pipeline.
- Fatal error and loader present; System page shows error message on load failure.

---

## Known limitations

- Not all Api.gs endpoints have been converted to the full envelope; migration can continue incrementally.
- Manual smoke (Start → Classic → SACRED → transactions, budgets, settings, system) recommended before promoting to stable.
